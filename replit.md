# Overview
This full-stack web application is a comprehensive consultation platform connecting consultants and clients, primarily leveraging an AI assistant for personalized financial insights. Its purpose is to enhance interactions, streamline financial guidance, and improve client outcomes and consultant efficiency. The platform includes a multi-tier subscription system for AI agents with revenue sharing, an employee licensing system, and a public landing page for lead generation, aiming to be a holistic solution for modern financial consultation.

# User Preferences
Preferred communication style: Simple, everyday language.
User requested "obsessive-compulsive" attention to detail when verifying what works and what doesn't.
User prefers direct SQL for any database modifications (not ORM migrations).

# System Architecture
The application features a modern UI/UX built with React 18, TypeScript, Vite, Tailwind CSS, and `shadcn/ui`, prioritizing accessibility and responsiveness. The backend uses Express.js, TypeScript, JWT, bcrypt, and PostgreSQL (Drizzle ORM), implementing robust role-based access control (consultant, client, super_admin) and a multi-profile system. The UI/UX employs a Revolut-style design system with reusable CSS utilities and shared components, ensuring responsive and accessible UI with proper dark mode support using CSS variables.

On hold, the inactivity timeout is paused via `sessionManager.pauseInactivityTimeout()` so the session stays alive; on unhold, the timeout resumes. AI audio output (`recordAudioOut`) also resets the inactivity timer to prevent premature timeout during AI speech. If an inactivity timeout does fire, a registered callback in `voice-bridge-server.ts` calls `handleCallStop()` + `uuid_kill` to properly hang up the FreeSWITCH call (instead of leaving the caller connected to silence). **Inbound Overflow on Server Rejection**: When VPS bridge passes an inbound call to Replit but Replit rejects it (CHANNELS FULL), the catch block now routes the call to the overflow queue (hold music + auto-return when slot opens) instead of killing it. Outbound calls still get `uuid_kill` on rejection (callback retry handles them). Overflow routing logic extracted into reusable `routeToOverflow()` function. **Outbound Hangup Callback System**: VPS bridge `esl-client.ts` intercepts CHANNEL_HANGUP events for outbound calls, maps FreeSWITCH hangup causes to status callbacks via `notifyOutboundCallResult()` in `caller-context.ts`. Mapping: NO_ANSWER/NO_USER_RESPONSE/RECOVERY_ON_TIMER_EXPIRE→`no_answer`, USER_BUSY/CALL_REJECTED→`busy`, NORMAL_CLEARING with effectiveDuration<15s→`short_call`. Guards: skips if AMD already detected, active Gemini session exists, or ORIGINATOR_CANCEL. Uses `effectiveDuration = billsec > 0 ? billsec : duration` for accuracy. **Attempts Log**: `scheduled_voice_calls.attempts_log` (JSONB array) populated by callback handler in all 3 branches (completed/failed/retryable) with `{attempt, status, hangup_cause, duration_seconds, timestamp, event}`. **Unified Contact Timeline**: `GET /api/voice/contact/:phone` returns `timeline[]` (voice_calls + scheduled_voice_calls + attempts_log entries + voice_call_events, sorted DESC) and `stats{}` (totalCalls, completed, failed, noAnswer, busy, shortCall, voicemail, totalDuration, avgDuration, firstContact, lastContact). **CRM Tab**: New tab in `consultant-voice-calls.tsx` with search/filter by contact type and last outcome, card-based contact list with inline stats, lazy-loaded expandable timeline per contact, color-coded event icons. **Enriched Contact Profile**: `consultant-voice-call-detail.tsx` enhanced with Timeline tab (first tab, vertical timeline with expandable call details), enriched stats header, attempts_log mini-timeline in Programmate tab, sidebar activity recap. **Configurable Backoff Mode**: `voice_retry_backoff_mode` (varchar: 'exponential' | 'manual') + `voice_retry_manual_delays` (jsonb array of minutes per attempt). In exponential mode: baseInterval × 2^attempt, cap 30min. In manual mode: user-defined delay per retry attempt (1-60 min each). UI shows mode selector (Esponenziale/Manuale), per-attempt input fields in manual mode, and live timeline preview for both modes. **Per-Number Max Concurrent Calls**: `max_concurrent_calls` is now per-number (stored in `voice_numbers.max_concurrent_calls`, default 5). Configured in the "Gestione Overflow" card per-number panel with an input (1-10). The Voice Bridge checks per-number limit BEFORE `createSession()` via `sessionManager.canAcceptCallForNumber()`. Global env var `MAX_CONCURRENT_CALLS` remains as safety cap. Overflow dequeue (`tryDequeueOverflow`) also respects per-number limits, re-inserting at queue front with preserved timeout when limits are full. **Inbound Overflow Queue**: When all concurrent AI lines are busy, inbound callers enter an overflow queue instead of being rejected. Per-number config in `voice_numbers` table: `overflow_enabled` (bool, default true), `overflow_timeout_secs` (int, default 120), `overflow_dtmf_enabled` (bool, default true — caller presses 1 to transfer to consultant), `overflow_auto_return` (bool, default true — auto-transfer back to AI when slot frees), `overflow_message` (text, custom TTS message). UI: "Gestione Overflow / Coda d'Attesa" card in VPS tab with switches, slider, input, textarea per number. CHANNEL_HANGUP cleanup in `esl-client.ts` removes disconnected callers from queue.

Audio files stored at `uploads/overflow-audio/{consultantId}/`. 10 audio slots: hold_A-E (rotating hold messages), position_prefix/suffix (queue position announcements), transferring, transfer_failed, timeout. Backend endpoints in `voice-router.ts`: generate via Gemini TTS (`POST /overflow-audio/generate`), manual upload (`POST /overflow-audio/upload`), list/delete/play/defaults. All audio auto-converted to 8kHz 16-bit mono WAV for FreeSWITCH via `convertWavTo8kMono()`. VPS download endpoints use service token auth (same as number-lookup). VPS `session-manager.ts` syncs audio to `/opt/sounds/overflow/{consultantId}/` with 24h cache and stale file cleanup. Dialplan `05_overflow_queue.xml` uses `${overflow_audio_dir}` channel variable (default: `/opt/sounds/overflow/default`). UI: "Configurazione" tab reorganized into 4 sub-tabs (Numeri & Voce, Coda d'Attesa, Chiamate Uscita, Connessione). "Audio Coda d'Attesa" card in Coda d'Attesa sub-tab with per-slot TTS generation, upload, preview, and delete.
*   **Scheda Contatto (Contact Profile):** Unified contact profile page at `/consultant/voice-calls/contact/:phone` aggregating call history, scheduled calls, Hunter data, proactive lead info, and retry status. API endpoint: `GET /api/voice/contact/:phone`. Also accessible via `/consultant/voice-calls/:id` (resolves callId to phone via `/api/voice/contact-by-call/:callId`). Features tabbed call history with accordion expansion, sidebar cards for profile/Hunter/lead/retry data.
*   **VoIP Number Provisioning (Inventory-Based Flow):** Two-level VoIP number management. **Admin** pre-purchases numbers from Telnyx API and stores them in `number_inventory` table. **Consultant** picks from available inventory (not Telnyx directly) and gets assigned instantly, then has 7 days for KYC documentation. **Architecture: Centralized IP Trunk** — all Telnyx numbers use a single global IP connection (`telnyx-ip` → `orbitale.sip.telnyx.com`). Config in `system_settings` (keys: `telnyx_api_key`, `telnyx_ip_connection_id`, `telnyx_webhook_public_key`). Service: `server/services/telnyx-provisioning.ts`. **DB tables**: `number_inventory` (admin-managed pool: available/assigned/reserved/released), `consultant_numbers` (number ownership, KYC status, deadline tracking), `voip_provisioning_requests` (KYC submission data), `voip_provisioning_documents` (uploaded docs). **Admin endpoints** (`server/routes/admin.ts`): `GET/POST/PATCH/DELETE /api/admin/number-inventory/*` — search Telnyx, purchase, list, release, update notes. **Consultant endpoints** (`server/routes/voice-router.ts`): `GET available-numbers` (reads from inventory), `POST select-number` (assigns from inventory, no Telnyx call), `GET my-number`, `POST upload-kyc-doc`, `POST submit-kyc-docs`, `DELETE kyc-doc/:id`. **Admin UI**: "Inventario Numeri VoIP" card in admin-settings VPS/Voice tab — search, bulk purchase, inventory table with status badges. **KYC mapping**: Requirements use `requirement_id` cross-referenced with global requirement_types. **KYC deadline enforcement**: Cron poller suspends numbers with expired KYC deadlines. **Form validation**: Backend validates fiscal code (16 alphanum), P.IVA (11 digits), email format, CAP (5 digits). Frontend shows inline errors.
*   **Delivery AI Agent (Dipendente Delivery):** AI-powered onboarding and delivery assistant at `/consultant/academy` (Delivery AI tab). Full-screen layout (bypasses max-w-7xl container when active). Three modes: **Onboarding** (consultant analyzes own business) **Discovery** (analyzes third-party client), and **Simulator** (test Luca by role-playing as a client — pick from 20 niches and 6 attitudes). Four phases: Discovery (9-phase deep conversation: Chi Sei → Clienti → Come Lavori → Comunicazione → Vendita → Formazione → Team → Obiettivi → **Dettagli Attività** (nome_attivita, sito_web, citta_operativa), targeting 20-30 messages) → Elaborazione (profile extraction with `[DISCOVERY_COMPLETE]` tag, rich JSON with fields for all 9 phases) → Report → Assistente (permanent delivery companion mode with package-aware guidance). Uses Gemini with thinking enabled via `getAIProvider()`. **Google Business Pre-Enrichment**: Before report generation, scrapes Google Maps via SerpAPI (rating, reviews, address) and website via Firecrawl (services, social links, team members) using keys from `superadminLeadScraperConfig` DB or env vars. Scraping has 15s/20s timeouts and fails silently. **Report generation is multi-step agent flow**: STEP 1 (Analysis + business intelligence) → STEP 2 (Draft) → STEP 3 (Review). **Rich report structure**: `lettera_personale`, per-package deep analysis with `punteggio` (1-10), `cosa_va_bene`, `cosa_non_funziona`, `come_correggere`, `diagnosi_critica`, `tabella_diagnostica`, `insight_chiave`, 4-week roadmap with KPIs, `azioni_questa_settimana` (3 priority actions), `chiusura_personale`. **Frontend**: Two-column layout with sticky sidebar chapter navigation (IntersectionObserver) and scrollable long-form document. Print CSS with page breaks. **PDF generation**: Server-side pdfkit endpoint at `GET /reports/:sessionId/pdf`. **Animated generation progress**: 5-phase progress indicator with timed transitions. DB tables: `delivery_agent_sessions`, `delivery_agent_messages`, `delivery_agent_reports`. Backend: `server/routes/delivery-agent.ts`. Prompt: `server/prompts/delivery-agent-prompt.ts`. Frontend: `client/src/components/delivery-agent/`.
*   **Skill Store:** A marketplace for importing and creating AI skills, which are markdown instructions injected into the Gemini AI system prompt. Supports file uploads and code execution.
*   **Lead Scraper:** A dual-engine search via SerpAPI (Google Maps and Google Search) with website enrichment via Firecrawl. Features smart caching, CSV export, CRM lead management with status tracking, notes, and deal values. Includes an AI Sales Agent for generating structured sales reports and AI Keyword Suggestions for search queries. A slide-in chat panel provides conversational interaction. **Search Modes**: 3 modes via segmented control — "Solo Cerca" (SerpApi only, no AI), "Predefinito" (search + Firecrawl + AI analysis), "Cerca + Outreach" (search + AI + auto-schedule outreach tasks). Mode stored in search metadata (`metadata.params.searchMode`). **CRM Outreach Filters**: Sidebar includes "Outreach" section filtering by task status (In Approvazione, Schedulati, In Corso, Contattati, In Trattativa, Non Raggiungibile) via LEFT JOIN with `ai_scheduled_tasks` on `outreachTaskId`. **Cascade Delete**: Deleting leads also removes associated outreach tasks from `ai_scheduled_tasks`. **Auto-Status Sync**: Lead `contactedChannels` and `leadStatus` are updated after actual delivery (not just task creation) for voice, WhatsApp, and email channels. Failed outreach sequences set `leadStatus` to `non_raggiungibile`. **Smart Location System**: Local Italian cities/regions database (~100 cities, 20 regions) with fuzzy matching (Levenshtein distance), aliases (province codes, English names), and population-weighted scoring. `searchItalianLocations()` provides instant autocomplete; `normalizeLocation()` auto-corrects typos and abbreviations (e.g., "mi"→"Milano", "milno"→"Milano", "toscna"→"Toscana"). Backend normalizes location before SerpAPI queries. Frontend shows structured suggestions (city + region) and "corretto" badge in confirm dialog when auto-corrected. Service: `server/services/lead-scraper-service.ts`. Endpoints: `GET /autocomplete/location`, `GET /normalize-location`.
