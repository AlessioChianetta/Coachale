Objective
Transform the Funnel Architect ("Architetto dei Funnel") from a standalone SSE chat into a proper AI employee (dipendente AI) that follows the exact same architecture as Marco, Alessia, Hunter, etc. Give it a name, register it in the AI roles system, use the agent_chat_messages table + processAgentChatInternal for 32K char context management, and integrate the employee-style chat into the Funnel Builder.

The Architetto should have access to Content Studio context — but the consultant chooses WHICH saved template (salvataggio) to inject. Each consultant can have multiple content_idea_templates with different niches, targets, and market research data. The FunnelChat UI will include a template selector so the consultant picks which one to use as context.

DB changes use raw SQL (no Drizzle migrations). A new column linked_template_id on ai_autonomy_settings.agent_contexts JSONB (under the architetto key) stores the selected template ID per consultant.
